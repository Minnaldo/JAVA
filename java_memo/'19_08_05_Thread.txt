1)스레드와 프로세스
 -프로세스 : 개별적으로 동작하는 프로그램 (이클립스, 메신저 등)
 -스레드 : 프로세스를 구성하는 작은 실행 단위 (메신저 = 채팅 + 파일 전송)
 -멀티스레드 : 동시에 여러 개의 프로세스를 실행하는 것.
 -멀티스레드 : 하나의 프로세스에서 여러 개의 스레드가 동시에 동작하는 것.

2) 멀티 스레드 프로그래밍의 장/단점
 : 장점 	- CPU 사용률 향상
	- 작업의 분리로 응답성 향상
	- 자원의 공유를 통한 효율성 증대
 : 단점	- 컨텍스트 스위칭 비용 발생
	- 스레드 제어의 어려움
